# 🔧 Build Error Fix - Step by Step

## ⚠️ Error:
```
error: package com.farmaa does not exist
    if (com.farmaa.BuildConfig.IS_NEW_ARCHITECTURE_ENABLED) {
```

## 🔍 Problem:
Autogenerated file me old package name `com.farmaa` reference hai. Build cache clean karna hoga.

## ✅ Solution:

### Step 1: Clean Build (PowerShell)

**Terminal me ye commands run karein:**

```powershell
# Farmaa/android folder me jao
cd Farmaa\android

# Clean build
.\gradlew clean

# Build folders delete karein
Remove-Item -Path "app\build" -Recurse -Force -ErrorAction SilentlyContinue
Remove-Item -Path "build" -Recurse -Force -ErrorAction SilentlyContinue

# Wapas root folder me jao
cd ..
```

### Step 2: Metro Cache Clean

**New Terminal:**
```powershell
cd Farmaa
npx react-native start --reset-cache
```

**Ya manually:**
```powershell
cd Farmaa
Remove-Item -Path "node_modules\.cache" -Recurse -Force -ErrorAction SilentlyContinue
```

### Step 3: Rebuild

**New Terminal:**
```powershell
cd Farmaa
npm run android
```

Autogenerated files regenerate honge with correct package name `com.furmaa`.
